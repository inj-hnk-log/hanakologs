# 🛡️ DCR士業ギルド構想：ユーザー評価 × スキルマッチング設計

タグ: DCR, 提言

### 🏰 ギルド概念の導入

- 士業を「ギルド」単位で束ね、**専門性・対応実績・人柄**で差別化
- ギルドは**分野ごと／地域ごと／クエスト傾向ごと**に設計可能
- 士業本人も複数ギルドに所属可能（兼業OK）

### 🌟 評価システムのポイント

| 機能 | 内容 |
| --- | --- |
| 🗣️ **ユーザー評価** | ★1〜5 ＋ 任意の短評（例：「説明がとても丁寧」「返信が早くて安心できた」） |
| 📊 **案件別スコア** | クエスト種別ごとの成功率、継続依頼率、報酬平均額など |
| 🧭 **ギルド内ランク** | 騎士団階級風にしても良し（例：Sage, Master, Knight, Novice） |
| 🛡️ **信頼バッジ制度** | 累計支援件数やギルド内協力実績に応じて付与される称号バッジ（例：黒帯・参謀・特攻隊長） |

### 🔁 ギルド×ユーザーのマッチングロジック（案）

- ユーザーが「●●系に強い先生がいい」「返信早めがいい」など条件指定すると、自動で推奨士業が提示される
- **評価スコア＋案件実績＋対応スピード**などを加味したレコメンド
- 条件に合致した士業が少ない場合は、ギルド全体に**一斉募集通知**（ギルド掲示板で見れる）

### 📦 ギルド参加士業にとってのメリット

| 項目 | 説明 |
| --- | --- |
| 📥 **案件流入** | ギルド単位で案件が入り、得意分野の仕事が増える |
| 🎖️ **可視化される信頼** | 評価・バッジで信頼度を示しやすく、営業不要 |
| 🧑‍🤝‍🧑 **仲間同士の横連携** | 分野外の相談をギルド仲間にパスしやすくなる |
| 🧾 **自己ブランディング** | ギルド経由で得た実績をSNSや履歴書でアピール可能 |
- “士業側のエコシステム”が整ってると、DCRの「一過性の優しさ」ではなく、**永続的に機能する支援構造**になる。
- 「いい先生に当たるかどうかは運」じゃなくなる。
