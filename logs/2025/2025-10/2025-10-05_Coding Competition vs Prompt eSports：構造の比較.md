# 🧠 Coding Competition vs Prompt eSports：構造の比較

タグ: ☆4.5, 知的インフラ, 社会, 観測記録

| 項目 | Coding Competition（例：AtCoder, LeetCode, Paiza） | Prompt eSports（例：JobsPlatform構想） |
| --- | --- | --- |
| 🎯 主目的 | アルゴリズムスキルの測定・実装能力の競争 | AIとの協働設計スキル・思考プロセスの評価 |
| 🛠️ 手段 | 人力でコードを書く、限られた時間で最適化 | AIにプロンプトで指示して成果を導く |
| 🧪 評価軸 | 正答率、速度、メモリ効率 | 最終成果＋プロセスの論理性・再現性・協働性 |
| 📜 提出物 | ソースコード1本 | プロンプト＋対話ログ＋成果物セット |
| 🤝 協働性 | 原則、単独作業 | AIペアプロ／チーム戦にも対応可能 |
| 🔁 再現性 | コード自体で担保（入力→出力） | プロンプト・AIとの対話ログが再現性の核 |
| 🧠 重視される力 | 実装速度・コーディング体力 | 課題解釈・指示設計・エラー対処力 |
| 💼 実務との接続 | Web系／SI系問わず汎用性あり | AIを使った業務効率化・提案能力と直結 |

---

## 🔍 考察：なぜPrompt eSportsはまだ主流でないのか？

- 従来の大会は“人力前提”文化の延長線
- AI利用が“チート”扱いされやすかった時代が長かった
- 対話ログの評価が難しい（主観性が入る）
- まだ教育機関や企業が評価指標を持ちきれていない

<img src="https://www.notion.so/icons/arrow-right_gray.svg" alt="https://www.notion.so/icons/arrow-right_gray.svg" width="40px" />

だが、**JobsPlatformのようにAI対話を可視化・構造化して提出する世界**では、
むしろこの対話ログこそが**新しい“履歴書”になる**。

</aside>

---

## 🧩 つながりのヒント：Paizaの再構成

はなの取り組み（Paiza演習をペアプロ化）も、実はこの**移行段階のモデル**。

- 従来：Aパート/Bパート構造で入力→出力の形式美を問う
- 現在：AIと一緒に「過程の意味」をログとして提出可能に
- 未来：ログそのものが“提案書”や“設計書”として価値を持つ

---

## 🏁 最終的なビジョン

> 「あなたはAIとどれだけ建設的に対話できるか？」
> 

この問いへのスコア化が、Prompt eSportsの本質。

コーディング大会が**実装体力の見本市**だったなら、

プロンプト競技は**構造知性と対話力の競技場**になる。
